Problem 2

(2a. number)
VALID
Unusual cases:
- Properties from built-in object Math (ie. Math.PI), Infinity, -Infinity, and NaN considered type number.
- Division of zero also returns number rather than undefined.
- Type coercion using math operations except addition returns number (ie. "6"-2 OR "3"*"3" OR 3+"7").
- Type coercion with number and boolean returns number.
- Operation on two boolean values return number.

(2b. int)
NOT VALID. JS number type includes integers.

(2c. unicode)
NOT VALID. Typeof unicode returns string.

(2d. list)
NOT VALID. Typeof list returns object.

(2e. map)
NOT VALID. Typeof map returns object.

(2f. class)
NOT VALID. Typeof class returns function so considered function type.

(2g. regex)
NOT VALID. Typeof regex returns object.

(2h. string)
VALID
Unusual cases:
- Typeof char and unicode is string.
- Type coercion: Adding string and number returns type string, not an error for operating on 2 different types.

(2i. boolean)
VALID
Unusual cases:
- When Boolean function is applied to other types, typeof returns type boolean. (ie. typeof(Boolean(1)) returns boolean)
- When using operators (&& or ||), they don't return boolean but the original operands (ie. "hello" && "123" returns "123" so type string)

(2j. object)
VALID
Unusual cases:
- Arrays, maps, and regex return as object.
- There are also some functions (ie.console.log) that return as object.
- And despite null being JS type null, their typeof is object.
- All constructor functions (ie. new String('')) return as object, EXCEPT for new Function().

(2k. null)
NOT VALID typeof output despite being a JS type
Unusual cases: Typeof null returns object even though its JS type is null.

(2l. function)
VALID despite not being JS type
Unusual cases: Although function technically belongs to JS object type, typeof returns function.
Typeof class and methods in objects is function.
Typeof new Function() is function, not object.

Problem 3

Declaring with "let" can limit variable to scope of a block so cannot be accessed from outside the block. While declaring with "var" allow variables to be accessed globally or locally in a function beyond the block scope.

Problem 4

(4a.)
Error Description:
- Comments are not supported in JSON.
- Values must be separated by commas.
- There should be no trailing commas.
Fixed Version: [1, 2, 3, 4, 5]

(4b.)
Error Description:
- Key and values in JSON objects are separated by colon, not dashes.
- Missing comma in between each JSON object.
- Last JSON object missing a curly brace at the end.
Fixed Version: [{ "name":"DDWA", "number": 3300 },{ "name": "Java", "number":2110 },{"name": "Python", "number": 1110}]

(4c.)
Error Description:
- Keys in JSON objects must be strings in double quotes.
- String values must also be in double quotes, not single quotes.
- Key-value pairs must be separated by commas.
Fixed Version: {"netid": "tk421","query": "do you copy?"}

(4d.)
Error Description:
- JSON Values must be of type string, number, null, or boolean, so (6-3) and undefined are not valid JSON values.
Fixed Version: [3, 4, 5.5, 6.5432, "(6 - 3)", null]
